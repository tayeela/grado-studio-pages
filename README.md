# ГРАДО Студия

Веб-САПР для градостроительных чертежей: границы территории, функциональные зоны,
красные линии, ЗОУИТ, здания, ТЭП. Работает в браузере, без установки и без сервера —
проект живёт в самой вкладке.

**Приложение:** https://tayeela.github.io/grado-studio-pages/

## Что умеет

**Черчение.** Полилинии, полигоны с дырами, дуги, окружности, точки, размерные линии.
Привязки к вершинам и рёбрам, шаг сетки, ввод длины с клавиатуры. Обрезка и продление
по границе, склейка линий, буфер, поворот, зеркало, масштаб.

**Подписи.** Подпись объекта ставится в самую «глубокую» точку контура (полюс
недоступности), поэтому у поймы, зоны вдоль набережной и квартала подковой она
остаётся внутри объекта, а не уезжает на соседа. Наложения разбираются по всему
чертежу сразу: спорное место достаётся объекту из верхнего слоя, а подпись шире
самого объекта не ставится вовсе. У текста ореол цветом фона — читается и поверх
заливки зоны, и поверх снимка. Подписывать можно точки и линии, не только
полигоны.

**Слои и знаки.** Слой заводится по требованию — быстрые клавиши создают или выбирают
слой нужного вида и сразу включают подходящий инструмент. Знаки ЛГР, категории внутри
слоя, условное форматирование, библиотека знаков проекта.

**Данные по видимой области.**

| источник | что берётся |
|---|---|
| OSM | дороги, здания, водные объекты, зелень |
| НСПД | земельные участки, ОКС, ЗОУИТ, границы |
| ГИС ОГД Москвы | функциональные зоны, красные линии (УДС, ТОП, ЛО, ОДМС), СЗЗ, водоохранные — и весь каталог портала, 663 слоя |

Геометрия чинится сразу при импорте: лишние точки, иглы, вырожденные части,
самопересечения. Красные линии портала приходят нарезанными на отрезки — студия
склеивает их в цельные линии по совпадению атрибутов, не проходя через перекрёстки.

**Определить объект.** Клик в этом режиме показывает все объекты под курсором
сверху вниз — зону, участок, здание и красную линию в одной точке. Наведение на
строку подсвечивает объект, клик выбирает его и делает активным его слой.

**Правка геометрии.** «Разрезать» — ломаная через объекты делит их на части,
атрибуты достаются каждой части; без выбора режется активный слой, с выбором —
только выбранное. «Объединить» собирает выбранные смежные полигоны одного слоя
в один объект (атрибуты от первого выбранного); разъединённые куски не
объединяются — модель чертежа не знает многочастных объектов. Разрез не теряет
и не добавляет площадь: части стыкуются без щели и без перекрытия.

**Символика по данным.** Кроме единого стиля и категорий по значению поля есть
градуированная символика: числовое поле (этажность, плотность, площадь) делится
на диапазоны — равными интервалами, по квантилям или по естественным границам
(Jenks) — и каждому диапазону достаётся свой цвет из палитры. Диапазоны со
счётчиками попадают в легенду. Толщина линии, размер знака и кегль подписи могут
считаться выражением от атрибутов (`if(этажность > 9, 2, 0.6)`) — как
data-defined в QGIS; значения кешируются до правки данных.

**Отбор объектов.** «Выбрать по выражению» (`этажность >= 5 и зона like 'Ж%'`)
и «Выбрать по расположению» (внутри границы, пересекает, содержит, не касается,
в радиусе N метров) — как в QGIS. Выборка складывается с уже набранной:
заменить, добавить, убрать, оставить общее. Выражения понимают сравнения,
`и` / `или` / `не`, `like`, `$area` и `$length`; те же выражения работают
в калькуляторе полей.

**Проверка топологии.** Слой проверяется на щели между полигонами, перекрытия,
дубликаты и самопересечения — как «Топологический чекер» в QGIS. Щель между
зонами не попадает ни в один ТЭП, перекрытие считается дважды, поэтому находки
показываются списком с переходом к месту на чертеже и подсвечиваются поверх
него. Проверка ничего не меняет в проекте. Порог перекрытия отсекает шум
оцифровки (по умолчанию 0,5 м²), порог щели — законные внутренние пустоты
(1000 м²). На 5000 полигонов все четыре проверки занимают около четверти секунды.

**ТЭП.** Площади и плотности по нормам Москвы, живой пересчёт, обрезка по границе
территории, учёт ограничений.

**Проект.** Сохранение в `.grado` (обмен с настольной версией и QGIS), автосохранение
в браузере, пять контрольных копий с восстановлением, отмена и повтор.

## Чего в браузерной версии нет

Выпуск DXF и печать в масштабе PDF требуют настольной версии — в меню «Выпуск» эти
пункты отключены и подписаны. Сохранение проекта, импорт и все инструменты черчения
работают полностью.

## Клавиши

`V` выбор · `A` дуга · `D` размер · `M` измерение · `T` обрезать · `E` продлить ·
`J` склеить · `R` поворот · `X` привязки · `Y` общие границы · `C` шаг сетки ·
`F` показать всё · `?` весь список

Быстрый слой: `G` граница · `Z` функциональная зона · `O` ограничение · `B` здание ·
`P` общественная зона · `L` красная линия · `S` соцобъект

## Доступность

Интерфейс проходится с клавиатуры целиком, включая обзор объектов чертежа: холст
берёт фокус, стрелки водят по объектам, PageUp/PageDown — по слоям, Enter читает
атрибуты. Диалоги удерживают фокус и возвращают его на кнопку-источник. Контраст
текста удовлетворяет WCAG 2.1 AA в обеих темах, движение выключается системной
настройкой «уменьшить движение».

## Разработка

Сборки нет: это статические файлы. Для локального запуска подойдёт любой файловый
сервер.

```bash
npx --yes http-server . -p 8231 -c-1
```

Тесты — обычные скрипты Node без фреймворка, каждый запускается сам по себе:

```bash
for f in tests/*.test.js; do node "$f"; done
```

Каждый тест начинается с описания дефекта, который он закрывает: тесты здесь —
это память о том, что уже ломалось.

### Как устроено

`index.html` подключает движок (`app*.js`) и слой браузерного бэкенда.
`pages-adapter.js` перехватывает `/api/*` и отвечает на них прямо во вкладке:
IndexedDB под проект и кэш слоёв, прямые запросы к Overpass, НСПД и ГИС ОГД.
`pages-core.js` — разбор и маршрутизация данных источников, `tokens.css` и
`redesign/*.css` — оформление.

Версия сборки лежит в `version.json`; страница сама сверяет её с зашитой в
`index.html` и перезагружается, когда выходит новая.
